# 产品需求文档 (PRD)

## Retro Color Palette Generator — 复古色卡提取工具

**版本：** 1.0  
**最后更新：** 基于当前实现整理

---

## 1. 产品概览

### 1.1 产品定位

一款面向设计初学者的**单页面 Web 应用**。用户上传图片后，可一键提取 5 种主色形成色板；通过「随机色相」在保持饱和度与亮度不变的前提下变换配色，获得无限配色灵感；支持一键复制色值，并配有系统日志与底部状态栏反馈。

### 1.2 核心价值

- **快速提取**：基于 Color Thief 与像素采样，提取 5 种视觉差异明显的代表色。
- **灵感扩展**：随机色相偏移（0–360°）同步作用于图片与色块，瞬间生成新配色。
- **治愈系视觉**：蓝、橙、绿温和配色，现代无衬线字体，圆角与柔和阴影，营造清新、温和、治愈的视觉氛围。

### 1.3 技术栈

- **前端**：原生 HTML + CSS + JavaScript
- **颜色提取**：Color Thief（CDN，`colorthief@2.3.2`）
- **字体**：Inter（Google Fonts），回退至 Roboto、system-ui

## 2. 用户流程

1. **进入页面** → 看到左右双栏：左侧为上传区，右侧为操作与色板区。
2. **上传图片** → 点击或拖拽图片至左侧上传区，支持 JPG / PNG / GIF。
3. **预览图片** → 上传成功后，左侧显示图片预览，右侧按钮与色板区保持可见。
4. **提取色板** → 点击「Extract Colors」，从当前图片中提取 5 种颜色并展示在右侧色板网格中。
5. **随机色相** → 点击「Randomize Hue」，图片与 5 个色块同步进行随机色相旋转（0–360°），系统日志记录偏移角度。
6. **重置** → 点击「Reset」，图片与色块恢复为最近一次提取的原始配色。
7. **复制色值** → 点击任意色块的 Hex 代码，复制到剪贴板，该处文字短暂变为「Copied!」约 1 秒后恢复，系统日志记录「Copied #xxxxxx」。

## 3. 界面布局 (UI Structure)

### 3.1 整体结构

- **页面背景**：整页柔和空气蓝，主内容在视口内**水平与垂直居中**。
- **主容器**：白色圆角卡片，最大宽度约 1100px，最小高度约 620px，带轻微阴影。
- **双栏比例**：左右各约 **50%**（flex: 1），保证左右视觉平衡。

### 3.2 左侧栏 — 图片上传与预览

- **标题**：主标题「Retro Color Palette Generator」，副标题「Upload Your Image」。
- **上传区（初始态）**
  - 虚线边框上传框，内含上传图标与「Click or Drag & Drop File」「JPG / PNG / GIF」提示。
  - 支持点击选择文件与拖拽上传；拖拽时边框与背景有视觉反馈（如薄荷绿高亮）。
- **预览区（上传后）**
  - 上传成功后隐藏上传区，显示「主内容」：深色边框的预览框，内嵌图片。
  - 图片自适应容器，保持比例，不变形。

### 3.3 右侧栏 — 操作与色板

- **标题**：主标题「Retro Color Palette Generator」，副标题「色卡提取」。
- **操作按钮组**（自上而下）
  - **Extract Colors**：从当前预览图提取 5 种颜色；无图片或未加载完成时点击会在系统日志中提示。
  - **Randomize Hue**：随机色相偏移；仅在已提取色板后可用（与 Reset 同步启用/禁用）。
  - **Reset**：恢复为最近一次提取的原始配色；同样在提取后可用。
- **色板区**
  - 标题「提取色板」。
  - 未提取时：占位文案「Click "Extract Colors" to analyze colors」。
  - 已提取时：**5 个色卡**以**网格**排列（如 3 列 × 2 行，小屏可 2 列或 1 列），每个色卡包含：
    - 色块（正方形或圆角矩形）；
    - Hex 代码（可点击复制）；
    - RGB 数值；
    - 悬停时显示该颜色在图片中的**占比百分比**（如 20%）。
- **系统日志**
  - 标题「System Log」。
  - 区域展示带时间戳的文本日志（如「[HH:MM:SS] Ready.」「[HH:MM:SS] Analyzing pixels...」「[HH:MM:SS] Hue shifted by 45deg.」「[HH:MM:SS] Copied #xxxxxx」），最近若干条（如 7 条），自动滚动。

### 3.4 底部状态栏

- 位于主容器下方，与主容器同宽（最大 1100px），圆角与主容器底部衔接。
- 深色背景（与右侧主色一致），白色文字，显示「Ready」等状态；使用与正文一致的现代字体。

## 4. 视觉设计规范 (Visual Design)

### 4.1 色彩方案（治愈系 · 蓝 / 橙 / 绿）

| 用途 | 色值 | 说明 |
|------|------|------|
| 页面背景 | `#E6F2F8` | 柔和空气蓝 (Soft Air Blue) |
| 左侧容器背景 | `#FAF8F5` | 极淡暖米白 (Warm off-white) |
| 左侧容器边框 / 主按钮(Extract) | `#E8B89A` | 温和珊瑚橙 (Gentle Coral Orange) |
| 右侧容器背景 | `#EEF6F4` | 极淡薄荷蓝白 (Mint-blue white) |
| 右侧容器边框 / Randomize / 状态栏 / 标题等 | `#7B8FA1`、`#5A6B7A` | 低饱和灰蓝 (Muted Slate Blue)，用于边框、按钮、标题文字 |
| 上传区虚线边框 / 点缀 | `#9EC9B5`、`#7BB89A` | 薄荷绿 / 鼠尾草绿 (Mint / Sage Green) |
| 复制成功提示 | `#6B9B7A` | 柔和鼠尾草绿 (Soft Sage Green) |
| Reset 按钮 | `#8B9DAF` | 略浅灰蓝 |

- **原则**：去除沉重紫/粉；整体温润、对比度适中，以蓝、橙、绿构成治愈系氛围。

### 4.2 字体与排版

- **字体**：Inter（400 / 500 / 600 / 700），回退 Roboto、system-ui。
- **标题**：约 1.25rem，字重 700，颜色为灰蓝（如 `#5A6B7A`）。
- **副标题 / 正文**：约 0.875–1rem，字重 500–600。
- **色卡 Hex**：约 1.1rem，字重 700，易于识别与点击。
- **RGB、百分比、日志**：约 0.85–0.95rem，保证可读性。

### 4.3 组件样式要点

- **按钮**：圆角矩形（如 10px），无像素化；Extract 为珊瑚橙，Randomize 为灰蓝，Reset 为浅灰蓝；悬停轻微上浮与阴影。
- **上传区**：虚线边框（如 3px dashed），悬停/拖拽时边框与背景使用薄荷绿系反馈。
- **色卡**：大而清晰的色块 + 下方 Hex/RGB；悬停时显示占比；整体为圆角卡片，带轻微边框与阴影。
- **底部状态栏**：与右侧主色一致的深色底，白字，现代字体。

## 5. 功能需求详述

### 5.1 图片输入与处理

| 需求项 | 描述 |
|--------|------|
| 上传方式 | 支持点击上传区唤起文件选择、以及拖拽文件到上传区释放。 |
| 支持格式 | 图片类型（如 JPG、PNG、GIF）。 |
| 预览行为 | 上传成功后，左侧仅显示图片预览框与图片，不再显示上传区。 |
| 状态重置 | 新图片加载后：清空当前色板展示、禁用 Randomize 与 Reset、清空/重置与色相相关的状态。 |

### 5.2 颜色提取

| 需求项 | 描述 |
|--------|------|
| 触发方式 | 用户点击「Extract Colors」按钮。 |
| 算法与库 | 使用 Color Thief 的 `getPalette(image, 5)` 获取 5 种颜色。 |
| 多样性保证 | 对提取结果按 RGB 欧氏距离过滤（如最小距离 40），不足 5 种时从原调色板中补足，确保 5 种颜色视觉区分明显。 |
| 占比计算 | 对图片像素进行采样（如降采样到约 320px 内长边），将每个像素归到最近的主色，统计各主色占比；百分比取整且总和为 100%。 |
| 结果展示 | 在右侧色板区以网格展示 5 个色卡，每个含色块、Hex、RGB、悬停显示占比。 |
| 日志 | 提取过程中输出「Analyzing pixels...」，成功输出「Palette extracted: 5 colors.」，失败输出「Failed to extract colors.»或「Extraction error. Try another image.»。 |

### 5.3 色板展示与交互

| 需求项 | 描述 |
|--------|------|
| 布局 | 5 个色卡以网格排列（如 3 列），每个色卡为独立卡片，内含色块与文字。 |
| 信息展示 | 每个色卡展示：Hex（如 #FFFFFF）、RGB（如 RGB(255,255,255)）、悬停时显示占比（如 20%）。 |
| 复制 | 点击 Hex 文本将对应 Hex 复制到剪贴板；该处文字变为「Copied!」，约 1 秒后恢复为原 Hex；复制成功/失败在系统日志中记录。 |

### 5.4 随机色相 (Randomize Hue)

| 需求项 | 描述 |
|--------|------|
| 触发方式 | 用户点击「Randomize Hue」按钮；仅在已成功提取色板后可用。 |
| 行为 | 生成 0–360 的随机整数作为色相偏移角度。 |
| 图片 | 对预览图应用 CSS `filter: hue-rotate(Ndeg)`，无动画，瞬间切换。 |
| 色块 | 对当前保存的 5 个原始色在 HSL 空间做相同角度的色相偏移，更新展示的 5 个色块与 Hex/RGB。 |
| 持久化 | 原始 5 色不变，仅展示层偏移；Reset 时基于原始 5 色恢复。 |
| 日志 | 输出「Hue shifted by Ndeg.»。 |

### 5.5 重置 (Reset)

| 需求项 | 描述 |
|--------|------|
| 触发方式 | 用户点击「Reset」按钮；仅在已提取色板后可用。 |
| 行为 | 移除图片的 hue-rotate 滤镜；将展示的 5 个色块与 Hex/RGB 恢复为最近一次提取的原始数据。 |
| 日志 | 输出「Reset to original palette.»。 |

### 5.6 系统日志与状态栏

| 需求项 | 描述 |
|--------|------|
| 系统日志 | 所有关键操作（就绪、加载图片、分析中、提取结果、色相偏移、重置、复制成功/失败）在 System Log 区域输出带时间戳的一行文本；保留最近若干条（如 7 条），超出则丢弃最旧。 |
| 底部状态栏 | 固定展示「Ready」等状态文案，样式与主色一致（深灰蓝底、白字）。 |

## 6. 技术约束与实现要点

### 6.1 结构约束

- 不改变现有 HTML 结构及元素 id，以保证现有 JavaScript 逻辑无需修改即可运行。
- 色卡由 JS 动态生成，类名（如 `.floppy`、`.floppy__hex`、`.floppy__color`、`.floppy__pct`、`.floppy__meta`、`.floppy__rgb`）需在 CSS 中保留使用，仅通过样式实现视觉呈现（如网格、圆角、字体、颜色）。

### 6.2 颜色与滤镜

- 色相偏移仅使用 CSS `filter: hue-rotate(deg)` 作用于预览图；色块数值由 JS 在 HSL 空间计算后写回 Hex/RGB。
- 颜色转换（RGB ↔ Hex ↔ HSL）在脚本内实现，不依赖额外库。

### 6.3 响应式

- 主容器最大宽度 1100px，小屏（如 ≤900px）可改为单列布局：上图为左栏，下图为右栏；色板网格可改为 2 列或 1 列，按钮可纵向堆叠。
- 所有交互与功能在桌面端优先保证，移动端保持可用与可读。

## 7. 验收标准 (Acceptance Criteria)

- [ ] 页面加载后，主应用在视口内水平与垂直居中，左右双栏约 50% 宽度，无内容被截断或布局错乱。
- [ ] 支持点击与拖拽上传图片，上传后左侧仅显示图片预览，右侧始终可见操作区与色板区。
- [ ] 点击「Extract Colors」能稳定提取 5 种颜色，色板区以网格展示 5 个色卡，每个含 Hex、RGB，悬停显示占比。
- [ ] 5 种颜色在视觉上差异明显（无严重重复或过于接近）。
- [ ] 点击「Randomize Hue」后，图片与 5 个色块同步发生色相变化，且系统日志中记录偏移角度。
- [ ] 点击「Reset」后，图片与色块恢复为最近一次提取的原始配色，日志记录重置。
- [ ] 点击任意色卡的 Hex 可复制到剪贴板，该处短暂显示「Copied!」约 1 秒后恢复，系统日志记录「Copied #xxxxxx」或失败信息。
- [ ] 系统日志与底部状态栏使用与设计规范一致的配色与字体，整体视觉为治愈系蓝/橙/绿，无沉重紫/粉感。
